Main Description

A cada miembro del grupo se le asigna un rol que es de interés para el sistema. Los roles son usuarios, el sistema mismo, otros sistemas y, a veces, entidades mantenidas por el sistema. Luego, el grupo explica cómo se utiliza el sistema. A lo largo del camino, habrá discusiones sobre quién es responsable de qué: tomar notas sobre las responsabilidades de cada función. Hacer que el analista de sistemas desempeñe el papel de usuario o cliente ayuda a obtener información real sobre el dominio del problema.

Como marco para el juego de roles, puede realizar tutoriales escritos sobre cómo se utiliza el sistema. Si tiene algunos casos de uso descritos, puede utilizarlos como base para el script. El tutorial también se puede realizar a nivel empresarial, utilizando los casos de uso empresarial como base para el script.

Otra técnica que a menudo se combina con el juego de roles son las tarjetas de colaboración de responsabilidad de clase (CRC).
