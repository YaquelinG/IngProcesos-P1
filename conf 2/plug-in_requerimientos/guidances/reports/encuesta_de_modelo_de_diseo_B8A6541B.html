Main Description

Este informe describe el modelo de diseño de manera integral, en términos de cómo se estructura el modelo en paquetes y qué clases hay en el modelo. Si está utilizando paquetes, el documento muestra la estructura del modelo jerárquicamente. El informe se puede utilizar para describir todo el modelo de diseño en diferentes etapas:

Durante la elaboración, como cuando hayas identificado las primeras clases y sus objetos.
Durante la construcción, cuando el diseño esté completo.
Este informe lo utilizan varias personas interesadas en el modelo de diseño, como el arquitecto de software, los diseñadores de casos de uso, los diseñadores, los evaluadores, los revisores y los administradores.

2 Jerarquía de diseño-modelo

Esta sección presenta los paquetes de diseño jerárquicamente, explica las dependencias entre ellos y muestra el contenido de cada paquete de forma recursiva.

Si el modelo tiene varios niveles de paquetes, los del nivel superior se presentan primero. Los paquetes dentro de estos se presentan a continuación, y así sucesivamente, hasta llegar a los paquetes en la parte inferior de la jerarquía. Para cada paquete incluye:

Su nombre .
Una breve descripción.
Una lista de las clases que posee el paquete, incluido el nombre y una breve descripción de cada clase.
Una lista de las relaciones que posee el paquete, incluido el nombre y una breve descripción de cada relación.
Una lista de los paquetes que pertenecen directamente al paquete, con cada paquete presentado de la misma manera jerárquica que antes.
