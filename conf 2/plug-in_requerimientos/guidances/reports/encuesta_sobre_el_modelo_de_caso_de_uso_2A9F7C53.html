Main Description

Este informe describe el modelo de casos de uso de manera integral, en términos de cómo se estructura el modelo en paquetes y qué casos de uso y actores hay en el modelo. Si está utilizando paquetes, el documento muestra la estructura del modelo jerárquicamente. El informe se puede utilizar para describir todo el modelo de caso de uso en diferentes etapas:

Durante el inicio, como cuando ha definido el alcance del sistema.
Durante la elaboración, como cuando el modelo de caso de uso es más estable.
Durante la construcción, cuando se completen los requisitos.
Este informe lo utilizan varias personas interesadas en el modelo de casos de uso, como el cliente, los usuarios, los arquitectos de software, los autores de casos de uso, los diseñadores, los diseñadores de casos de uso, los evaluadores, los administradores, los revisores y los escritores.

3. La jerarquía del modelo de casos de uso
Esta sección presenta los paquetes de casos de uso jerárquicamente, explica las dependencias entre ellos y muestra el contenido de cada paquete de forma recursiva. Si el modelo tiene varios niveles de paquetes, los del nivel superior se presentan primero. Los paquetes dentro de estos se presentan a continuación, y así sucesivamente, hasta llegar a los paquetes en la parte inferior de la jerarquía. Para cada paquete incluye:

Su nombre.
Una breve descripción que explica la función y el papel del paquete en el sistema. La descripción debe ser comprensible para cualquier desarrollador que quiera utilizar el paquete.
Una lista de los casos de uso que posee el paquete, incluido el nombre y una breve descripción de cada caso de uso.
Una lista de actores propiedad del paquete, incluido el nombre y una breve descripción de cada actor.
Una lista de relaciones propiedad del paquete, incluido el nombre y una breve descripción de cada relación.
Una lista de los paquetes que pertenecen directamente al paquete, con cada paquete presentado de la misma manera jerárquica que antes.
