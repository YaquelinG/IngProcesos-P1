Tool Mentor: Detallar un caso de uso utilizando Rational Requisite Pro
Este mentor de herramientas describe cómo utilizar Rational RequisitePro® para describir en detalle un caso de uso del sistema. La descripción del caso de uso del sistema la realiza el analista del sistema.

Main Description

Una vez identificados los casos de uso para el sistema propuesto (como se describe en el Mentor de herramientas de Rational Rose®: Búsqueda de actores y casos de uso ), puede utilizar RequisitePro para desarrollar un documento de especificación de caso de uso.

Nota: Puede desarrollar los casos de uso en Rose y generarlos en RequisitePro utilizando la función Gestión integrada de casos de uso. Consulte  Tool Mentor: Gestión de casos de uso con Rational Rose y Rational RequisitePro para obtener más información.

Las secciones del documento de especificación de casos de uso se pueden utilizar para crear requisitos específicos. Estos requisitos pueden rastrearse (o vincularse) a otros requisitos, como las características del producto.

La información textual para los casos de uso seleccionados la detalla un especificador de requisitos que escribe una Especificación de Caso de Uso para cada caso de uso. Este documento define todas las propiedades textuales del caso de uso y puede detallar el nombre y la descripción del caso de uso producido en la actividad de RUP Buscar actores y casos de uso .

Pasos de la herramienta
Para detallar un caso de uso usando RequisitePro:

Crear un documento de especificación de caso de uso
Complete el documento de especificación de casos de uso
Cree requisitos en la especificación detallada de casos de uso
Agregar diagramas a la especificación de casos de uso (opcional)
Si creó su proyecto de RequisitePro utilizando la plantilla de caso de uso, su proyecto ya contiene un tipo de documento de especificación de caso de uso.

1. Cree un documento de especificación de caso de uso
El documento de especificación de caso de uso contiene las propiedades textuales del caso de uso. Esto incluye las siguientes propiedades del caso de uso: nombre, breve descripción, flujo básico de eventos, flujo alternativo de eventos, condiciones previas, condiciones posteriores y requisitos especiales.

Nota: Si ha desarrollado sus casos de uso en Rose, puede utilizar los procedimientos descritos en  Tool Mentor: Gestión de casos de uso utilizando Rational Rose y Rational RequisitePro para crear un nuevo documento de caso de uso asociado con su caso de uso de Rose. De lo contrario, utilice los siguientes pasos de la herramienta para crear un documento de caso de uso.

Para crear un documento de especificación de caso de uso en RequisitePro:

En el Explorador, seleccione el paquete en el que desea que resida el nuevo documento. Luego haga clic en Archivo > Nuevo > Documento . Aparece el cuadro de diálogo Propiedades del documento.
Escriba un nombre, una descripción y un nombre de archivo para el documento.
En el cuadro de lista desplegable Tipo de documento , seleccione "Tipo de documento de especificación de caso de uso".
Haga clic en Aceptar . El esquema del documento de especificación de caso de uso se abre en Microsoft® Word.
Para más información
 Consulte el tema titulado Creación de documentos de requisitos (Índice: documentos>creación) en la ayuda en línea de RequisitePro.

2. Complete el documento de especificación de casos de uso.
En el documento de especificación de caso de uso recién creado, escriba información específica del caso de uso en cada sección. El nombre y las propiedades de descripción breve ya deberían haberse documentado en Actividad: Buscar casos de uso y actores en Rational Rose.

Para completar el documento de especificación de casos de uso:

Reemplace el texto "Nombre del caso de uso" que se muestra en el esquema con el nombre real del caso de uso.
Nota: Si creó el documento de caso de uso utilizando los procedimientos descritos en Tool Mentor: Gestión de casos de uso utilizando Rational Rose y Rational RequisitePro , el nombre del caso de uso se inserta automáticamente en el título del documento. Utilice los comandos RequisitePro > Requisito > Cortar y Pegar para mover el requisito del caso de uso al texto "Nombre del caso de uso".

Lea las instrucciones en el campo Descripción breve , luego elimínelas y escriba una breve descripción.

Nota: Si desarrolló el caso de uso en Rose y desea incluir el campo de documentación de Rose como parte de la sección de descripción breve en su documento de caso de uso de RequisitePro, copie el texto del campo Documentación en el cuadro de diálogo Especificación de caso de uso de Rose y péguelo en su documento de especificación de caso de uso.
Reemplace el texto predeterminado ubicado en la sección Flujo básico de eventos con el texto del flujo básico de eventos de este caso de uso. Identifique cada paso en una línea separada.
Repita este procedimiento para las otras propiedades del caso de uso (flujo alternativo de eventos, requisitos especiales, condiciones previas, condiciones posteriores, etc.).
Haga clic en RequisitePro > Documento > Guardar .
Para más información
 Consulte el tema titulado Guardar documentos de requisitos (Índice: documentos>guardar) en la ayuda en línea de RequisitePro.

3. Cree requisitos en la especificación detallada de casos de uso.
Para realizar un seguimiento de los vínculos de trazabilidad entre los casos de uso y otra información, como el modelo de caso de uso o las características del producto, cree requisitos de RequisitePro a partir de las secciones de Especificación de casos de uso. Haga que el nombre del caso de uso sea un requisito principal y cree requisitos secundarios a partir de sus propiedades. Estas propiedades pueden incluir descripciones breves, acciones dentro del flujo básico o alternativo de eventos, condiciones previas, condiciones posteriores, requisitos especiales y relaciones extendidas.

En el documento Especificación de caso de uso, seleccione el nombre del caso de uso.
Haz una de las siguientes:
Haga clic derecho y seleccione Crear requisito .
Haga clic en RequisitePro > Requisito > Nuevo .
Aparece el cuadro de diálogo Propiedades del requisito.
Seleccione UC como tipo de requisito.
En la pestaña Atributos , seleccione el valor del atributo de Propiedad de " Nombre " de la lista desplegable de propiedades de casos de uso.
Repita los pasos anteriores para la descripción breve (estableciendo el atributo de Propiedad en " Descripción breve "). En la pestaña Jerarquía , seleccione <elegir padre> e identifique el requisito de UC que representa el nombre del caso de uso.
En la sección de flujo básico de eventos del documento Especificación de casos de uso, cree requisitos de UC para cada paso o grupo de pasos (subflujo) al que desee establecer enlaces de trazabilidad. Establezca el atributo Propiedad en Flujo básico y establezca el requisito principal en el nombre del caso de uso creado en los pasos 1 a 3 anteriores. Tienes la opción de indicar grupos de pasos que siempre se realizan juntos. Si es necesario, utilice requisitos jerárquicos para distinguir los subflujos del flujo básico de eventos.

Nota: No es necesario crear requisitos para cada paso de un flujo de eventos. El valor de crear requisitos de flujo de eventos está en rastrear desde un requisito de nivel superior, como una característica del producto, hasta una parte específica del flujo de eventos. Un flujo de eventos a menudo abarca varias páginas, por lo que puede ser preferible considerar el flujo de eventos como un todo.
En cada flujo alternativo de eventos, cree requisitos de UC para cada paso o grupo de pasos (subflujo) al que desee establecer vínculos de trazabilidad. Establezca el atributo de Propiedad en Flujo alternativo y el requisito principal como se indicó anteriormente. De manera similar al flujo básico de eventos, utilice requisitos jerárquicos para indicar subflujos completos.
Los siguientes pasos son opcionales:
En la sección de condiciones previas del documento Especificación de caso de uso, seleccione cada condición previa por separado y cree un requisito de UC (Propiedad = condiciones previas , padre = requisito de nombre de caso de uso).
Repita el paso anterior para las condiciones posteriores (Propiedad = Condiciones posteriores ) y la sección de requisitos especiales (Propiedad = Especial ). Establezca el requisito de nombre del caso de uso como padre.
Para más información
 Consulte el tema Creación de requisitos en un documento (Índice: requisitos>creación) en la Ayuda en línea de RequisitePro.

4. Agregue diagramas a la Especificación de casos de uso (opcional)
Algunas de las propiedades de los casos de uso no son textuales, como los "diagramas de casos de uso" y "otros diagramas". Consulte Artefacto RUP : caso de uso . Estos diagramas se almacenan en Rose. Con Rational SoDA®, puede crear un informe de caso de uso a partir de las propiedades textuales del caso de uso almacenadas en RequisitePro y la información del diagrama de caso de uso almacenada en Rose. Consulte Informe: Caso de uso sobre cómo crear este informe.
