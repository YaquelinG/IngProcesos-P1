Main Description

Objetivo
Este mentor de herramientas describe la captura y gestión de solicitudes de partes interesadas utilizando las diversas funciones proporcionadas por Rational ClearQuest.

Información relacionada con el proceso unificado de Rational: Actividad: Obtener solicitudes de las partes interesadas .

Descripción general
Rational ClearQuest ayuda al analista a recopilar eficazmente las solicitudes de las partes interesadas al proporcionar un formulario de solicitud de mejora completo y coherente. Este formulario es totalmente personalizable y puede diseñarse para su lugar de trabajo de desarrollo. Todas las solicitudes se registran en una única base de datos central de solicitudes de cambios. Las partes interesadas internas (miembros del equipo) utilizan la aplicación web o Windows ClearQuest para enviar solicitudes de mejora; las partes interesadas externas pueden utilizar la aplicación web ClearQuest. Una vez enviadas las solicitudes de mejora, el analista puede calificar aún más estas solicitudes agregando información a cada solicitud.

Este documento contiene los siguientes pasos:

1. Registre solicitudes de mejora mediante ClearQuest Web
ClearQuest proporciona una aplicación web que se puede configurar para proporcionar funcionalidad restringida a licenciatarios que no son de ClearQuest. Con acceso restringido, un usuario web de ClearQuest puede enviar solicitudes de cambio y se le proporciona acceso a una única consulta de ClearQuest definida por el administrador. Esta característica única permite a las partes interesadas clave que no pertenecen al equipo de software que construye el sistema utilizar la Web para ingresar sus comentarios. Además. Estas partes interesadas permanecen informadas del progreso de la solicitud y recuperan el estado actualizado ejecutando la consulta de acceso limitado.

Para presentar una solicitud de mejora mediante ClearQuest Web:

Inicie su navegador y escriba la URL web de ClearQuest. Aparece la página de inicio de sesión de ClearQuest.
Escriba su nombre de usuario y contraseña , luego haga clic en el botón Iniciar sesión . Ahora está conectado a la base de datos de ClearQuest en la Web.
En la lista de la Tabla de contenido en el lado izquierdo de la pantalla, ubique el encabezado Operaciones y haga clic en el subtítulo Enviar un registro . Aparece la página Seleccionar tipo de registro .
Haga clic en la flecha hacia abajo en el campo Seleccionar tipo de registro y aparecerá una lista de tipos de registros disponibles. Seleccione Solicitud de mejora y haga clic en el botón Enviar . El formulario Enviar solicitud de mejora aparece dividido en tres secciones.
En la sección Principal , escriba información en los campos Título , Origen , Prioridad del cliente y Descripción .
En la sección Adjunto del registro, adjunte cualquier información de respaldo disponible para la solicitud de mejora haciendo clic en el botón AGREGAR y vinculándolo al archivo externo.
En la tercera sección de Información de contacto , ingrese toda la información aplicable del cliente y del remitente . (nombre, empresa, dirección de correo electrónico).
Haga clic en Aceptar . La solicitud de mejora se crea y se agrega a la base de datos central.
Para obtener información detallada sobre la configuración de ClearQuest Web para proporcionar funcionalidad restringida a licenciatarios que no son de ClearQuest, consulte Edición de la configuración web en la ayuda en línea de ClearQuest Web.

2. Organizar solicitudes de mejora en ClearQuest Windows
Una vez enviada la solicitud de mejora, ClearQuest proporciona campos adicionales organizados por pestañas para complementar la información enviada originalmente en el formulario de registro. La pestaña Análisis , la pestaña Resolución y la pestaña Notas permiten al analista equilibrar la información del remitente con información específica del equipo, como la prioridad de marketing y el lanzamiento objetivo, para fines organizativos. El analista utiliza esta información para consultar la base de datos de solicitudes de mejora para determinar qué solicitudes se implementarán en cada versión.

Para organizar solicitudes de mejora utilizando ClearQuest Windows:

Seleccione la solicitud de mejora y haga clic en Acción > Modificar en el lado derecho del formulario de Solicitud de mejora. Esto activa la solicitud y le permite editar el registro.
Haga clic en la pestaña Análisis para activarla.
Establezca valores para cada uno de los campos relacionados haciendo clic en la flecha hacia abajo en cada campo ( Prioridad de marketing, Tipo de solicitud, Producto, Área de producto, Lanzamiento objetivo y Propietario ) y seleccionando los valores deseados. Nota : Estos campos son completamente personalizables; Los campos especificados se proporcionan de forma inmediata en ClearQuest.
Cuando haya terminado de editar, haga clic en Aplicar para guardar los cambios.
Las solicitudes de mejora ahora se pueden consultar según los valores establecidos.

Para obtener información sobre consultas y personalización de campos, consulte la ayuda en línea de ClearQuest.

3. Ejecutar consultas
ClearQuest proporciona una potente interfaz de consulta que permite al analista consultar todos los defectos y solicitudes de mejora en el proyecto conectado. Las siguientes son consultas que el analista puede encontrar útiles para gestionar las solicitudes de mejora. El analista podría ejecutar las siguientes consultas:

Aísle aquellas solicitudes de mejora destinadas a la versión específica que se aborda. Debido a que no todas las solicitudes de mejora recibidas se pueden implementar en la versión actual, resulta útil consultar todas las solicitudes de mejora utilizando el campo "Versión objetivo". Esta consulta devuelve una lista de trabajo de todas las solicitudes de mejora que se han clasificado y designado para la versión indicada en la consulta.
Enumere todas las solicitudes de mejora que no se han asociado con un requisito. Esta consulta ayuda al analista a ver aquellas solicitudes de mejora que aún están bajo consideración o en la cola, pero que aún no se han asignado a una iteración de software específica.
Localice el número de apariciones o instancias de la misma solicitud de mejora. Esta consulta ayuda al analista a determinar cuántas veces las partes interesadas han enviado una solicitud de mejora. Esto se suma al proceso de ponderación para determinar la importancia de la solicitud para las partes interesadas y el nivel de prioridad.
Determine cuántas solicitudes son verdaderas solicitudes de mejora para la funcionalidad existente frente a aquellas que determinan una nueva funcionalidad. Esta consulta muestra el progreso del desarrollo del software. Aquí el analista puede discernir si la mayoría de las solicitudes de mejora son para corregir la funcionalidad existente o si requieren la adición de nuevas características al producto actual.
Una vez que el analista ha aislado las solicitudes de mejora de la versión actual aprobadas, se vinculan fácilmente con requisitos válidos en la base de datos de requisitos mediante la integración de ClearQuest y RequisitePro.
