      Steps
      Asignar atributos

      El Plan de Gestión de Requisitos define los atributos a los que se les dará seguimiento para cada tipo de requisito. Los atributos más importantes son el beneficio (desde la perspectiva de las partes interesadas), el esfuerzo para implementar, el riesgo para el esfuerzo de desarrollo, la estabilidad (probabilidad de permanecer sin cambios) y el impacto arquitectónico (si es arquitectónicamente significativo) de cada requisito.

      El beneficio y la estabilidad los establece el analista del sistema, en consulta con las partes interesadas. El esfuerzo y el riesgo los establece el director del proyecto, en consulta con el arquitecto de software. El impacto arquitectónico lo establece el arquitecto de software.

      Los requisitos inestables con alto riesgo, alto esfuerzo o alto beneficio deben señalarse para realizar más análisis. Los requisitos de beneficios bajos con alto esfuerzo, riesgo o inestabilidad deben señalarse para su posible eliminación.

      Establecer y verificar la trazabilidad
      El Plan de gestión de requisitos define cómo se rastrean los tipos de requisitos hasta otros artefactos. El Analista de Sistemas debe establecer la trazabilidad requerida y utilizar periódicamente informes de trazabilidad para garantizar que la trazabilidad se mantenga de acuerdo con el Plan de Gestión de Requisitos.
      Gestionar requisitos cambiantes

      Los cambios de requisitos se gestionan de acuerdo con el Plan de Gestión de Requisitos. Algunas pautas adicionales son las siguientes:

      Reevaluar los atributos de requisitos y la trazabilidad
      Incluso si un requisito no ha cambiado, los atributos y la trazabilidad asociados con un requisito pueden cambiar. El analista de sistemas es responsable de mantener esta información de forma continua.

      Gestionar el cambio jerárquicamente
      Un cambio en un requisito puede tener un efecto dominó que afecte a otros requisitos, diseño u otros artefactos relacionados. Para gestionar este efecto, debes cambiar los requisitos de arriba hacia abajo. Revise el impacto en la Visión , luego el Modelo de Caso de Uso , el Modelo de Diseño y el Material de Soporte al Usuario Final . Para gestionar el impacto del cambio de requisitos en el esfuerzo de prueba, revise la información relacionada en Actividad: Definir necesidades de trazabilidad y evaluación . Los informes de trazabilidad son útiles para determinar los elementos potencialmente afectados.
