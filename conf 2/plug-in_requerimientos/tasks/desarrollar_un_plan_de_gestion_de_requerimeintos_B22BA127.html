Task: Desarrollar un plan de gestión de requerimientos
Purpose
Desarrollar un plan para documentar los requisitos, sus atributos y lineamientos para la trazabilidad y gestión de los requisitos del producto.

    Main Description

    Se debe desarrollar un Plan de Gestión de Requisitos para especificar la información y los mecanismos de control que se recopilarán y utilizarán para medir, informar y controlar los cambios en los requisitos del producto.

    Antes de comenzar a describir los requisitos del proyecto , debe decidir cómo documentarlos y organizarlos, así como también cómo utilizar los atributos de requisitos al gestionar los requisitos a lo largo del ciclo de vida del proyecto.

    Elegir los atributos y la trazabilidad adecuados para los requisitos de su proyecto le ayudará a:

    Evaluar el impacto del proyecto de un cambio en un requisito.
    Evaluar el impacto del fracaso de una prueba en los requisitos (es decir, si la prueba falla, es posible que no se cumpla el requisito).
    Gestionar el alcance del proyecto.
    Verificar que todos los requisitos del sistema se cumplan mediante la implementación.
    Verifique que la aplicación haga solo lo que estaba destinada a hacer.
    Gestionar el cambio.
    Documente todas las decisiones relativas a documentos de requisitos, elementos de trazabilidad (consulte trazabilidad y tipos de requisitos ), directrices y estrategias para los atributos de requisitos en el Plan de gestión de requisitos.

    Steps
    Establecer trazabilidad

    Primero debe identificar los elementos de trazabilidad entre los cuales desea establecer vínculos de trazabilidad. Los elementos de trazabilidad más importantes y la trazabilidad típica entre ellos se describen en Conceptos: Trazabilidad.

    El resultado se documenta en un conjunto de matrices de trazabilidad de requisitos, que forman parte del artefacto Atributos de requisitos.

    Elija atributos de requisitos
    Los atributos se utilizan para realizar un seguimiento de la información asociada con un elemento de trazabilidad, normalmente con fines de estado y generación de informes. Los atributos esenciales a rastrear son Riesgo, Beneficio, Esfuerzo, Estabilidad e Impacto Arquitectónico , para permitir priorizar requisitos para la gestión del alcance y asignar requisitos a iteraciones.
    Mapa a herramientas

    La trazabilidad y los atributos son conceptos generales que pueden aplicarse a cualquier artefacto o elemento de artefacto. Sin embargo, un proyecto típico tendrá herramientas de programación y presupuestación, herramientas de diseño, herramientas de gestión de requisitos y herramientas de gestión de configuración. Estas herramientas a menudo proporcionan y/o imponen ciertos atributos y trazabilidad.

    Por ejemplo, las herramientas de programación suelen proporcionar vínculos entre personas y tareas, y gestionar atributos como el porcentaje de finalización. El vínculo entre la tarea y el requisito puede estar implícito mediante una convención de nomenclatura o puede gestionarse explícitamente.

    Las herramientas de diseño, como Rational Rose, proporcionan vínculos entre elementos de diseño mediante el lenguaje de modelado unificado (UML) y administran atributos como "Descripción", "Persistencia", etc.

    Los siguientes mentores de herramientas brindan orientación para vincular información entre herramientas:

    Gestión de casos de uso utilizando Rational Rose y Rational RequisitePro
    Gestión de solicitudes de partes interesadas utilizando Rational ClearQuest y Rational RequisitePro

    Escribe el plan
    El Artefacto: Plan de gestión de requisitos describe los aportes necesarios para un plan eficaz. La plantilla pretende servir como guía. La intención de cada sección debe abordarse dentro del contexto de un proyecto/producto determinado. Las pautas detalladas se proporcionan en Pautas: Plan de gestión de requisitos.
