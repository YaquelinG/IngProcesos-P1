Task: Modelar interfaz de usuario
Purpose
Construir un modelo de interfaz de usuario que respalde el razonamiento y la mejora de su usabilidad

      Steps
      Describir las caracteristicas de los actores relacionados
      Describir las características de los actores (humanos) relacionados con el caso de uso. Concéntrese en describir el actor principal del caso de uso, ya que la mayor parte de la interacción involucra a este actor. Esta información es importante para los pasos siguientes.
      Crear un guión gráfico de caso de uso
      Comience creando un guión gráfico del caso de uso y asigne una dependencia de seguimiento entre los dos. El guión gráfico del caso de uso tiene varias propiedades, como campos de texto y diagramas, que describen los aspectos de usabilidad del caso de uso.
      Describir el flujo de eventos: guión gráfico

      La descripción paso a paso de cada caso de uso que se incluye en esta actividad debe perfeccionarse y centrarse en cuestiones de usabilidad; Este refinamiento se captura en la propiedad Flujo de eventos - Guión gráfico del guión gráfico del caso de uso.

      Comience creando un esquema inicial del flujo de eventos: guión gráfico utilizando breves declaraciones de acción, para adquirir una comprensión del caso de uso.

      Luego complemente el Flujo de Eventos - Guión Gráfico con la guía deseada, valores y volúmenes promedio de objetos, y usos de acciones promedio para las diferentes partes del flujo de eventos.

      Capture los requisitos de usabilidad en el guión gráfico del caso de uso
      Capture los requisitos de usabilidad en el guión gráfico del caso de uso, definiendo qué tan alta debe ser la usabilidad de la interfaz de usuario.
      Encuentre las clases de límites necesarias para el guíon gráfico del caso de uso

      En este paso identificamos las clases de límites centrales y primitivas necesarias para proporcionar la interfaz de usuario del caso de uso.

      Identificar clases de límites centrales
      Identifique una clase de límite central que represente la ventana principal con la que interactúa el actor. Si dicha clase ya existe, considere reutilizarla. Esto se debe a que debe esforzarse por minimizar la cantidad de ventanas principales con las que un actor en particular necesita interactuar. Sin embargo, por supuesto, tendrá que hacer concesiones; Las ventanas principales no deben ser demasiado complejas ni abarrotadas.

      Estas clases de límites centrales son a menudo agregados de clases de límites más primitivas (ver más abajo).

      Ejemplos de clases de límites centrales:
      La clase de límite central identificada en una solicitud de correo es Mail Box. Para obtener más información, consulte Directrices: Clase de límite .

      La clase de límite central identificada en un editor de documentos es Documento. Para obtener más información, consulte Directrices: Clase de límite .

      Identificar clases de límites primitivas
      Estudie el flujo de eventos: guión gráfico en detalle y cualquier otra descripción de caso de uso disponible. Busque información interesante para el usuario e información que deba ser visible y administrada en la interfaz de usuario.

      Considere cualquier objeto o término de entidad comercial (especialmente sustantivos) en un glosario. Considere también los objetos de entidad si existen. Éstas son formas de comprender qué información manipula el sistema y, por tanto, también resulta interesante para el usuario.

      Luego, cree una clase de límite para cada objeto lógico manipulado por el actor a través de la interfaz de usuario. Al hacer esto, considere siempre reutilizar clases de límites ya existentes. El resultado de este paso preparará para la construcción de una interfaz de usuario orientada a objetos.

      Ejemplos de clases de límites primitivas:
      Las clases de límites primitivas identificadas en una aplicación de correo incluyen el archivo adjunto de mensaje de correo. Para obtener más información, consulte Directrices: Clase de límite .

      Las clases de límites primitivas identificadas en un editor de documentos incluyen Párrafo y Nota al pie. Para obtener más información, consulte Directrices: Clase de límite .

      Describa las clases de límites identificadas
      Describe cosas obvias sobre las clases de límites. En esta etapa, a menudo es posible delinear la mayoría de sus atributos, agregaciones, asociaciones y también algunas de sus responsabilidades. Recuerde que las clases de límites centrales son a menudo agregaciones de clases de límites más primitivas.

      Cada clase se perfeccionará y describirá en detalle en los pasos siguientes; consulte los pasos a continuación para ver ejemplos.

      Relacionar las clases de límites con el guión gráfico del caso de uso
      Las clases de límites que participan en un guión gráfico de caso de uso deben estar relacionadas con él. Esto se hace capturando las clases en diagramas de clases, incluidas sus relaciones, adjuntos a los guiones gráficos de los casos de uso.

      Describir las interacciones entre objetos límite y actores

      Este paso es opcional y se realiza si la secuencia de interacción (por ejemplo, flujo de ventanas y rutas de navegación) necesita capturarse formalmente en un diagrama de interacción.

      Recorra el flujo de eventos del caso de uso (o más precisamente, el subconjunto del caso de uso bajo consideración en la iteración actual), identificando los objetos límite responsables del comportamiento visible del caso de uso. Tenga en cuenta que estos objetos de límite son instancias de las clases de límite que se encuentran en el paso anterior (Buscar clases de límite necesarias para el guión gráfico del caso de uso). Ilustre las interacciones entre los objetos de límite participantes creando uno o más diagramas de colaboración.

      El diagrama de colaboración también debe mostrar las interacciones entre el sistema y sus actores. El flujo de eventos generalmente comienza cuando uno de los actores solicita algo al sistema, ya que un actor siempre invoca el caso de uso.

      Ilustrar las interacciones entre objetos creando vínculos entre ellos; anota los enlaces con una breve descripción de la intención del objeto invocador, es decir, lo que quiere lograr en la interacción con el objeto invocado. Estas "intenciones" luego se integrarán en las responsabilidades de cada clase de límite correspondiente, como se describe en el siguiente paso (Describir las responsabilidades de las clases de límite).

      Complemente los diagramas del guión gráfico del caso de uso
      Este paso es opcional y se realiza si los diagramas y descripciones (como se crearon anteriormente) deben complementarse y aclararse aún más con referencias a un prototipo de interfaz de usuario ya existente (si corresponde).
      Consulte el prototipo de interfaz de usuario del guión gráfico del caso de uso
      Este paso es opcional y se realiza si los diagramas y descripciones (como se crearon anteriormente) deben complementarse y aclararse aún más con referencias a un prototipo de interfaz de usuario ya existente (si corresponde).
