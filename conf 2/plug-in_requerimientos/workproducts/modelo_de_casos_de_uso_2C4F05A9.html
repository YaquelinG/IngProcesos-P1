Artifact: Modelo de casos de uso
El modelo de caso de uso es un modelo de las funciones previstas del sistema y su entorno, y sirve como contrato entre cliente y los desarrolladores. El modelo de caso de uso se utiliza como entarda esencial para las actividades de análisis, diseño y pruebas.
Available States: solo actores, refinado, reestructurado
Purpose

Las siguientes personas utilizan el modelo de casos de uso:

El cliente aprueba el modelo de caso de uso. Cuando tienes esa aprobación, sabes que el sistema es lo que el cliente quiere. También puede utilizar el modelo para analizar el sistema con el cliente durante el desarrollo.
Los usuarios potenciales utilizan el modelo de casos de uso para comprender mejor el sistema.
El arquitecto de software utiliza el modelo de casos de uso para identificar funcionalidades arquitectónicamente significativas.
Los diseñadores utilizan el modelo de casos de uso para obtener una descripción general del sistema. Cuando perfecciona el sistema, por ejemplo, necesita documentación sobre el modelo de caso de uso para ayudar en ese trabajo.
El gestor utiliza el modelo de casos de uso para planificar y dar seguimiento al modelado de casos de uso y también al diseño posterior.
Las personas ajenas al proyecto pero dentro de la organización, los ejecutivos y los comités directivos, utilizan el modelo de casos de uso para obtener una idea de lo que se ha hecho.
Las personas revisan el modelo de casos de uso para brindar comentarios adecuados a los desarrolladores de forma regular.
Los diseñadores utilizan el modelo de casos de uso como base para su trabajo.
Los evaluadores utilizan el modelo de casos de uso para planificar las actividades de prueba (pruebas de casos de uso y de integración) lo antes posible.
Quienes desarrollarán la próxima versión del sistema utilizarán el modelo de casos de uso para comprender cómo funciona la versión existente.
Los redactores de documentación utilizan los casos de uso como base para escribir las guías de usuario del sistema.
