Artifact: Visión
La visión define la visión de los stakeholders sobre el producto a desarrollar, especifica en terminos las necesidades y las caracteristicas clave de los stakeholders. Al contener un resumen de los requisitos básicos previstos, proporcionan la base contractual para lso requisitos técnicos más detallados.
Available States: refinado
Purpose

El documento de Visión proporciona una base de alto nivel, a veces contractual, para los requisitos técnicos más detallados. También puede haber una especificación de requisitos formales. La Visión captura requisitos de muy alto nivel y restricciones de diseño para brindar al lector una comprensión del sistema a desarrollar. Proporciona información al proceso de aprobación del proyecto y, por lo tanto, está íntimamente relacionado con el Caso de Negocio. Comunica los "por qué y qué" fundamentales relacionados con el proyecto y es un indicador con el que se deben validar todas las decisiones futuras.

El documento de Visión será leído por los gerentes , las autoridades de financiación que desempeñan funciones en el modelado de casos de uso y los desarrolladores en general.

Description
Main Description

La visión de un proyecto debe poder modificarse a medida que evoluciona la comprensión de los requisitos, la arquitectura, los planes y la tecnología. Sin embargo, debería cambiar lenta y normalmente durante la primera parte del ciclo de vida.

Es importante expresar la visión en términos de sus casos de uso y escenarios principales a medida que se desarrollan, para que pueda ver cómo los casos de uso realizan la visión. Los casos de uso también proporcionan una base eficaz para desarrollar un conjunto de casos de prueba.

El autor original puede ser cualquiera, pero cuando se establece el proyecto, la Visión es responsabilidad del analista del sistema.

Otro nombre utilizado para este documento es Documento de requisitos del producto.
