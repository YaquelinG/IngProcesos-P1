Guideline: Revisar los requisitos existentes
Relationships
Related Elements
  • Elicitar requerimientos de los stakeholders
Main Description

Es posible que tenga especificaciones de requisitos de sistemas anteriores o relacionados de otro modo como referencia; puede resultar útil analizarlas. O es posible que haya comenzado a utilizar Rational Unified Process algún tiempo después de que comenzara el proyecto.

Con el grupo, analice cada requisito para encontrar comportamientos de aplicaciones o atributos de comportamiento. En general, durante el recorrido, debe ignorar la información explicativa, como introducciones y descripciones generales del sistema.

Mantenga una lista de todos los problemas que identifique y asegúrese de que alguien tenga la tarea de resolver cada problema. Es posible que deba hacer algunas suposiciones si un requisito no está claro. Lleve un registro de estas suposiciones para poder verificarlas con las partes interesadas.

Recuerde quién escribió los requisitos. Busque posibles "requisitos fuera de lugar", es decir, cosas que están fuera del alcance del proyecto. Si no sabe si algo es un requisito, pregunte a las partes interesadas.

Es muy eficaz realizar este tipo de tutoriales utilizando cualquier esquema de caso de uso existente como marco. Cada requisito debe estar relacionado con al menos un caso de uso en su esquema. Si no hay ningún caso de uso con el que relacionarse, es una indicación de que falta un caso de uso o que el requisito está fuera de lugar.
