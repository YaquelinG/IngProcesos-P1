Tool Mentor: Estructuración del modelo de casos de uso utilizando Rational Rose
Este mentor de herramientas describe cómo utilizar Rational Rose para documentar las relaciones entre actores y entre casos de uso.

Main Description

A continuación se muestra un resumen de los pasos que se realizan para documentar las relaciones entre actores y entre casos de uso:

Documentar generalizaciones de actores

Documentar relaciones de inclusión entre casos de uso
Documentar relaciones extendidas entre casos de uso
Generalizaciones de casos de uso de documentos
Para obtener información detallada sobre los diagramas de casos de uso, consulte:

1.Documentar generalizaciones de actores
Puede insertar una relación de generalización entre un actor empresarial y otro actor empresarial en un diagrama de casos de uso utilizando la herramienta de dibujo Generalización de la caja de herramientas del diagrama de casos de uso. Una vez que haya creado la relación, puede describirla agregando texto al campo de documentación del cuadro de diálogo Generalizar especificación.

2.Documentar relaciones de inclusión entre casos de uso
Para crear una relación de inclusión entre dos casos de uso en un diagrama de casos de uso, primero crea una dependencia entre los dos casos de uso y luego asigna un estereotipo de inclusión a la dependencia. Se utiliza el estereotipo de inclusión cuando un caso de uso emplea la funcionalidad de otro caso de uso. El caso de uso que se utiliza normalmente contiene funcionalidades que otros casos de uso pueden necesitar o desear.

Una vez que haya creado la dependencia entre casos de uso, puede describir la relación agregando texto al campo Documentación de la Especificación de dependencia.

3.Documentar relaciones extendidas entre casos de uso
Para crear una relación de extensión entre dos casos de uso en un diagrama de casos de uso, primero cree una dependencia entre los dos casos de uso y luego asigne un estereotipo de extensión a la dependencia. El estereotipo extendido se utiliza para expresar un comportamiento opcional o condicional para un caso de uso.

Una vez que haya creado la asociación entre casos de uso, puede: describir la relación agregando texto al campo Documentación de la Especificación de dependencia.

4.Generalizaciones de casos de uso de documentos
Puede crear una relación de generalización de un caso de uso empresarial a otro caso de uso cuando un caso de uso proporciona una funcionalidad común (por ejemplo, cuando tiene un caso de uso abstracto que proporciona una funcionalidad común a casos de uso concretos).

Una vez que haya creado una relación de generalización entre casos de uso, puede describir la relación agregando texto al campo Documentación de la Especificación de generalización.
