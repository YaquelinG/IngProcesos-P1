Artifact: Actor
Un actor define un conjunto coherente de roles que los usuarios del sistema pueden desempeñar al interactuar con él. Una instancia de actor puede ser interpretada por un sistema individual o externo.
Available States: caracterizado, descrito brevemente
Purpose

Diferentes partes interesadas utilizan este artefacto para diferentes propósitos:

Analistas de sistemas : para definir los límites del sistema.
Diseñadores de interfaces de usuario : para capturar características de actores humanos.
Autores de casos de uso : para describir casos de uso y su interacción con los actores.
Analistas de objetos : para realizar casos de uso y su interacción con los actores.

Description
Main Description

Los actores se encuentran y se relacionan con los casos de uso en las primeras etapas de la fase de inicio, cuando se determina el alcance del sistema. Las características de los actores se describen antes de crear el prototipo e implementar la interfaz de usuario.

Responsabilidad
El analista de sistemas es el responsable último de gestionar a los actores. Aunque tanto el especificador de requisitos como el diseñador de interfaz de usuario actualizarán la información detallada sobre cada actor, el analista del sistema es responsable de garantizar que:

Cada actor tiene las comunicaciones-asociaciones correctas con los casos de uso en los que participa.
Cada actor es parte de las relaciones de generalización correctas.
Cada actor define un rol cohesivo y es independiente de otros actores.
Un diseñador de interfaz de usuario es responsable de la integridad de los detalles del actor humano, asegurando que:

Cada actor captura las características necesarias para construir la interfaz de usuario.
Los diagramas de casos de uso locales que describen al actor son legibles y coherentes con las demás propiedades.
Sastrería
Decide qué propiedades usar y cómo usarlas. En particular, debe decidir con qué nivel de detalle se debe describir la propiedad "Características".
