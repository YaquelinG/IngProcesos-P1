Artifact: Especificaciones suplementarias
Las especificaciones suplementarias capturan los requisitos del sistema que no se capturan fácilmente en los casos de uso del modelo de casos de uso.
Available States: esquemas, detallada
Purpose

Los siguientes roles y conjuntos de roles utilizan las especificaciones complementarias:

Los analistas crean y mantienen las Especificaciones complementarias, que sirven como medio de comunicación entre el analista, el cliente y los desarrolladores.
Los desarrolladores utilizan las Especificaciones Suplementarias como referencia al definir responsabilidades, operaciones y atributos en las clases, y al ajustar las clases al entorno de implementación.
Los implementadores consultan las Especificaciones suplementarias para obtener información al implementar clases.
Los gerentes consultan las Especificaciones complementarias para obtener información al planificar iteraciones.
Los probadores utilizan las Especificaciones suplementarias para verificar el cumplimiento del sistema.

Description
Main Description

Las Especificaciones Suplementarias van de la mano con el modelo de casos de uso , lo que implica que:

son considerados inicialmente en la fase de Inception , como complemento a la definición del alcance del sistema
se refinan de manera incremental durante las fases de Elaboración y Construcción .
Responsabilidad
Un analista de sistemas es responsable de producir las Especificaciones Suplementarias, que son un complemento importante para el modelo de casos de uso. Las Especificaciones Suplementarias y el modelo de casos de uso juntos deberían capturar un conjunto completo de requisitos del sistema.

Sastrería
Los tipos de requisitos complementarios varían ampliamente entre proyectos, por lo que se debe aplicar una adaptación para definir las secciones aplicables a su proyecto. Decida qué información (atributos) incluir en la Visión y cuál gestionar utilizando herramientas de gestión de requisitos, como Rational RequisitePro (consulte Guía de herramientas: Gestión de dependencias utilizando Rational RequisitePro® ), y también consulte Artefacto: Atributos de requisitos.
