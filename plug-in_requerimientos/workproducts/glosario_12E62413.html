Artifact: Glosario
El glosario define terminos importantes utilizados por el proyecto
Available States: refinado
Purpose

Existe un glosario para el sistema que proporciona un conjunto coherente de definiciones para ayudar a evitar malentendidos. Los miembros del proyecto utilizan inicialmente el Glosario para comprender los términos específicos del proyecto. Este documento también es importante para las personas que desempeñan estas funciones:

Desarrolladores , que utilizan los términos del Glosario al diseñar e implementar clases, tablas de bases de datos, interfaces de usuario, etc.
Analistas , que utilizan el Glosario para capturar términos específicos del proyecto para que puedan definir claramente las reglas comerciales y garantizar que las especificaciones de requisitos hagan un uso correcto y consistente de esos términos.
Desarrolladores de cursos y redactores técnicos , que utilizan el glosario para elaborar material de capacitación y documentación utilizando terminología reconocida.

Description
Main Description

Momento
El Glosario se desarrolla principalmente durante las fases de inicio y elaboración, porque es importante acordar una terminología común al principio del proyecto.

Responsabilidad
El rol de Analista de Sistemas es responsable de la integridad del Glosario, asegurando que:

se produce en el momento oportuno
se mantiene continuamente consistente con los resultados del desarrollo
Sastrería
En algunos proyectos en los que no se realizan modelos de negocio ni de dominio, el glosario es el artefacto principal utilizado para capturar información sobre el dominio de negocios del proyecto.

Si el contexto y el alcance del esfuerzo de modelado de negocios es mucho más amplio que el del esfuerzo de ingeniería de software, es posible que necesite producir un glosario separado, específicamente para el modelado de negocios. Ese Glosario sería entonces responsabilidad del Analista de Procesos de Negocio.
