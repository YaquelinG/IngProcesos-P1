Artifact: Guión gráfico de casos de uso
Un guión gráfico de caso de uso es una descripción lógica y conceptual de cómo la interfaz de usuario proporciona un caso de uso, incluida la interacción requerida entre los actores y el sistema.
Purpose

Las siguientes personas utilizan los guiones gráficos de casos de uso:

diseñadores de interfaces de usuario , para construir un modelo de interfaz de usuario;
diseñadores de los objetos límite que participan en el guión gráfico del caso de uso, para comprender las funciones de los objetos en los casos de uso y cómo interactúan los objetos. Usan esta información para diseñar e implementar el objeto límite (es decir, para construir la interfaz de usuario);
aquellos que diseñan la próxima versión del sistema para comprender cómo el sistema lleva a cabo el flujo de eventos en términos de objetos límite. Por ejemplo, un cambio puede afectar a un número limitado de casos de uso, en cuyo caso los diseñadores necesitan ver la realización de su flujo de eventos;
aquellos que prueban para probar los casos de uso del sistema;
el gerente para planificar y dar seguimiento al trabajo de análisis y diseño.

Description
Main Description

Los guiones gráficos de casos de uso se producen tan pronto como se priorizan sus casos de uso correspondientes para considerarlos desde una perspectiva de usabilidad. El guión gráfico de casos de uso se realiza antes de crear un prototipo e implementar la interfaz de usuario (es decir, tanto en la disciplina de requisitos como en la disciplina de análisis y diseño).

Responsabilidad
Un diseñador de interfaz de usuario es responsable de la integridad del guión gráfico del caso de uso y garantiza que:

el flujo de eventos: el guión gráfico es legible y se adapta a su propósito;
los diagramas que describen el guión gráfico del caso de uso son legibles y se adaptan a su propósito;
los Requisitos de Usabilidad sean legibles y se ajusten a su propósito, y que capturen correctamente los requisitos de usabilidad del caso de uso correspondiente en el modelo de caso de uso;
la dependencia del seguimiento con el caso de uso correspondiente en el modelo de caso de uso es correcta;
las relaciones, como las asociaciones de comunicación, las relaciones de inclusión y extensión, del caso de uso correspondiente en el modelo de caso de uso se manejan correctamente dentro del guión gráfico del caso de uso.
Recomendamos que el diseñador de interfaz de usuario responsable de un guión gráfico de caso de uso también sea responsable de las clases de límites y las relaciones empleadas en el guión gráfico de caso de uso.

Sastrería
Decida si los guiones gráficos de casos de uso son útiles para su proyecto. Adaptarse para respaldar las necesidades del proyecto. Esto puede incluir incluir solo un subconjunto de subartefactos (propiedades), adaptar el nivel de formalidad en el que se crean y gestionan los subartefactos, y adaptar los subartefactos individuales. Documente las decisiones de adaptación en Artefacto: Directrices de diseño.
