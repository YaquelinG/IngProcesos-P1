Artifact: Modelo de diseño
El modelo de diseño es un modelo de objetos que describe la realización de casos de uso y sirve como una abstracción del modelo de implementación y su código fuente. El modelo de diseño se utiliza como entrada esencial para las actividades de implementación y prueba.
Purpose

El modelo de diseño es una abstracción de la implementación del sistema. Se utiliza para concebir y documentar el diseño del sistema de software. Es un artefacto compuesto e integral que abarca todas las clases de diseño, subsistemas, paquetes, colaboraciones y las relaciones entre ellos.

Description
Main Description

El modelo de diseño establece principalmente la arquitectura, pero también se utiliza como vehículo de análisis durante la fase de elaboración. Luego se perfecciona mediante decisiones de diseño detalladas durante la fase de construcción. El modelo de diseño se mantiene continuamente coherente con el modelo de caso de uso y el modelo de implementación.

Responsabilidad
Un arquitecto de software es responsable de la integridad del modelo de diseño, asegurando lo siguiente:

El modelo de diseño en su conjunto es correcto, consistente y legible. El modelo de diseño es correcto cuando realiza la funcionalidad descrita en el modelo de caso de uso, y sólo este comportamiento.
La arquitectura en el modelo de diseño cumple su propósito, incluidas las vistas lógica, de proceso y de implementación. Estas vistas se recopilan en un artefacto separado; consulte Artefacto: documento de arquitectura de software .
Tenga en cuenta que el arquitecto de software no es responsable de los paquetes, clases, relaciones, realizaciones de casos de uso ni de los diagramas mismos; en cambio, estos están bajo las responsabilidades de los diseñadores correspondientes y del diseñador de casos de uso.
