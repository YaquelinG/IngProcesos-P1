Artifact: Plan de iteración
Un conjunto de actividades y tareas secuenciadas en el tiempo, con recursos asignados, que contienen dependencias de tareas, para la iteración; un plan detallado.
Purpose

Las siguientes personas utilizan el Plan de iteración:

El director del proyecto , para planificar las tareas y actividades de iteración, programar las necesidades de recursos y realizar un seguimiento del progreso con respecto al cronograma.
Miembros del equipo del proyecto , para comprender qué deben hacer, cuándo deben hacerlo y de qué otras actividades dependen.

Description
Main Description

El plan de iteración para la próxima iteración se planifica en la iteración actual. Se modifica según sea necesario durante la iteración.

Un plan de iteración se introduce en el siguiente plan de iteración. Un Plan de Iteración queda obsoleto después de la iteración.

Responsabilidad
El Gerente de Proyecto es responsable de la integridad de los Planes de Iteración.

Sastrería
El Plan de Iteración debe detallar lo que se debe hacer de manera detallada, de modo que haya poco espacio para la confusión sobre la verdadera posición o responsabilidades en cualquier momento. Generalmente se utilizará algún tipo de herramienta de planificación de proyectos (como Microsoft® Project).

información adicional
Este es un plan detallado para una iteración. A menudo hay dos planes de este tipo: uno para la iteración actual y otro en construcción para la siguiente iteración.

Para definir el contenido de una iteración necesita:

el plan del proyecto
el estado actual del proyecto (en camino, retrasado, gran cantidad de problemas, requisitos progresivos, etc.)
una lista de escenarios o casos de uso que deben completarse al final de la iteración
una lista de riesgos que deben abordarse al final de la iteración
una lista de cambios que deben incorporarse en el producto (correcciones de errores, cambios en los requisitos)
una lista de clases o paquetes principales que deben implementarse completamente
Estas listas deben estar clasificadas. Los objetivos de una iteración deben ser agresivos para que, cuando surjan dificultades, se puedan eliminar elementos de las iteraciones según sus rangos.

Criterios de evaluación
Cada iteración concluye con una evaluación . Para esta evaluación de iteración, usted evalúa los resultados de la iteración en relación con los criterios de evaluación que se establecieron para el Plan de iteración.

Los criterios de evaluación se establecen antes de cada iteración y establecen objetivos para el conjunto de características, la calidad y el rendimiento que se alcanzarán en la iteración. El logro real de estos objetivos variará. Por ejemplo, en una iteración determinada, es posible que se supere el conjunto de funciones, que apenas se logre la calidad y que falte el rendimiento.

Además, las metas pueden expresarse como metas mínimas y deseables. Por ejemplo, puede haber un conjunto de características requeridas y algunas características deseables que se intentarán en esta iteración si la velocidad de desarrollo y los niveles de personal lo hacen factible.
