Artifact: Prototipo de interfaz de usuario
Un prototipo de interfaz de usuario es un prototipo de interfaz de usuario. Por ejemplo, el prototipo puede manifestarse como:
* bocetos o imágenes en papel
* mapas de bits de una herramienta de dibujo
* un prototipo ejecutable interactivo; por ejemplo, en Microsoft®Visual Basic®
Purpose

Los siguientes roles utilizan el prototipo de interfaz de usuario:

especificadores de requisitos , para comprender la interfaz de usuario de un caso de uso
analistas de sistemas , para comprender cómo la interfaz de usuario impacta el análisis del sistema
diseñadores , para comprender cómo impacta la interfaz de usuario y qué requiere desde el "interior" del sistema
aquellos que prueban las clases , para planificar actividades de prueba

Description
Main Description

Puedes hacer tres tipos básicos de prototipos:

Dibujos (en papel)
Mapas de bits (herramienta de dibujo)
Ejecutables (interactivos)
En la mayoría de los proyectos, debes utilizar los tres prototipos, en el orden indicado anteriormente.

Momento
El prototipo de interfaz de usuario se construye tempranamente, durante el inicio o al comienzo de la fase de elaboración, y antes de que se analice, diseñe e implemente todo el sistema (incluida su interfaz de usuario "real").

Tenga en cuenta que el objetivo principal de crear un prototipo de interfaz de usuario es poder exponer y probar tanto la funcionalidad como la usabilidad del sistema antes de que comience el diseño y desarrollo real. De esta manera, puede asegurarse de que está creando el sistema correcto, antes de dedicar demasiado tiempo y recursos al desarrollo.

Para lograr esto, de las pruebas iniciales se desprende que el desarrollo del prototipo debe ser significativamente más barato que el sistema real, y al mismo tiempo debe tener capacidades suficientes para poder soportar una prueba de uso significativa.

Responsabilidad
Un diseñador de interfaz de usuario es responsable de la integridad del prototipo de interfaz de usuario, asegurando que el prototipo contribuya a una interfaz de usuario utilizable de acuerdo con los requisitos de los guiones gráficos de casos de uso y los objetos límite.

Sastrería
Decida si un prototipo es adecuado para su proyecto. Decida si el prototipo incluye dibujos, mapas de bits y/o un ejecutable interactivo. Decida qué parte de la interfaz de usuario creará un prototipo y la profundidad y el realismo de cualquier interactividad. Decida si el prototipo es puramente desechable o si se pretende que algunos aspectos evolucionen hasta convertirse en el producto final.
